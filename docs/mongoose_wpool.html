    mongoose_wpool.start_sup_pool/3
    this function starts the worker_pool's pool under a specific supervisor in MongooseIM application. It's needed for 2 reasons: 1. We want to have a full control of all the pools and its restarts 2. When a pool is started via wpool:start_pool it's supposed be called by a supervisor, if not, there is no way to stop the pool.

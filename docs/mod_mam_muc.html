    mod_mam_muc behaviour (MongooseIM v4.0.1)

    XEP-0313: Message Archive Management

    The module uses several backend modules:

    • Preference manager (mod_mam_muc_rdbms_prefs);
    • Writer (mod_mam_muc_rdbms_arch or mod_mam_muc_rdbms_async_pool_writer);
    • Archive manager (mod_mam_muc_rdbms_arch);
    • User's ID generator (mod_mam_muc_user).

    Preferencies can be also stored in Mnesia (mod_mam_mnesia_prefs). This module handles MUC archives.

    This module should be started for each host. Message archivation is not shaped here (use standard support for this). MAM's IQs are shaped inside shaper_srv.

    Message identifiers (or UIDs in the spec) are generated based on:

    • date (using timestamp());
    • node number (using ejabberd_node_id).

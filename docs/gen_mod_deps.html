    Specs

    gen_mod_deps.resolve_deps/2

    Determines all modules to start, along with their parameters.

    NOTE: A dependency will not be discarded during resolving, e.g. if the resolver processes dependencies in order:

    deps(mod_a, []) -> [{mod_b, []}, {mod_c, []}] deps(mod_parent, []) -> [{mod_a, [param]}]

    then the dependency for mod_a will be reevaluated with new parameters and might return:

    deps(mod_a, [param]) -> [{mod_c, []}]

    In this case, mod_b will still be started.
